# 数字李白 (Digital Li Bai)

> 与诗仙对话，学唐诗，聊人生 —— 基于 LLM 知识库方法论的 AI 角色扮演对话系统。

---

## 项目概览

**数字李白**是一个基于大语言模型（LLM）的 AI 对话应用。它以唐代诗人李白（701-762）为数字人格，用户可以通过自然语言与"李白"对话——教诗、论诗、聊人生、即兴创作。

项目的核心设计理念源自 [Karpathy LLM Wiki 方法论](https://github.com/karpathy/llm-wiki)：通过结构化知识库（时间线、实体关系、主题分类、意象库）实现**按需加载上下文**，而非一次性塞入所有知识，从而在有限的上下文窗口内达到最佳的检索增强生成（RAG）效果。

### 核心能力

| 模式 | 触发方式 | 加载知识 | 李白行为 |
|------|----------|----------|----------|
| 教诗模式 | 提及具体诗名，问释义 | 诗文全文 + 创作背景 + 时期/主题 | 第一人称讲解，引导学生自主体会 |
| 论诗模式 | 问风格、技巧、创作方法 | 主题分类 + 时期风格对照 | 谈论创作理念、体裁偏好 |
| 人生模式 | 问生平、经历、友人 | 完整时间线 + 人物关系网 | 以故事形式回顾心路历程 |
| 创作模式 | 要求即兴作诗 | 相关诗作风格参照 + 意象库 | 以李白风格作诗（标注 AI 模拟） |
| 闲聊模式 | 轻松话题、比较性问题 | 人物关系 + 性格关键词 | 李白式幽默与豪气聊天 |

```

           │ message + history
           ▼
┌──────────────────────┐
│   context_builder.py │  ← 核心引擎：意图分类 + 知识检索 + 上下文组装
│   (Context Engine)   │

│  │ timeline  │ │ entities │ │  themes   │   │
│  │  时间线   │ │ 实体关系 │ │ 主题分类  │   │
│  └───────────┘ └──────────┘ └───────────┘   │
│  ┌───────────┐ ┌──────────────┐              │
│  │  imagery  │ │context-router│              │

│  └───────────┘ └──────────────┘              │
└─────────────────────────────────────────────┘
           │
           ▼
┌──────────────────────┐
│   OpenAI API         │  ← 模型推理（默认 gpt-4o）
│   (LLM Inference)    │
└──────────┬───────────┘
           │
           ▼
      李白的回复
```

### 上下文组装流程

1. **意图分类** — 正则匹配将用户输入归入 `teach_poem / discuss_poetry / life / create / chat` 五类之一
2. **知识检索** — 根据意图从 wiki 知识库和 poems 诗集中按需加载相关片段
3. **上下文组装** — 将 system prompt + 检索到的知识片段拼接为结构化上下文（XML 标签包裹）
4. **LLM 推理** — 发送组装好的上下文给 OpenAI API，生成李白风格的回复

## 核心模块详解

### `app.py` — 对话界面

基于 **Gradio** 构建的 Web 聊天界面，负责：

- 用户对话的输入/输出管理
- 调用 `context_builder.build_context()` 构建知识上下文
- 调用 OpenAI Chat Completions API 生成回复
- 保持最近 10 轮对话历史

启动方式：

```bash
python app.py [--port 7860] [--share]
```

### `context_builder.py` — 上下文引擎

这是整个系统的核心，包含四个关键组件：

**1. 意图分类器 `classify_intent()`**

基于正则表达式的意图识别，将用户输入归类为 5 种模式之一：
- `teach_poem` — 教诗/解诗（匹配《》、赏析、解读等）
- `discuss_poetry` — 论诗（匹配风格、技巧、体裁等）
- `life` — 聊人生（匹配经历、故事、当官、流放等）
- `create` — 即兴创作（匹配作诗、创作、即兴等）
- `chat` — 默认闲聊

**2. 诗歌检索引擎**

- `find_poem_by_title()` — 模糊标题匹配
- `find_poems_by_topic()` — 关键词全文搜索
- `get_poems_by_period()` — 按时期筛选
- `extract_titles_from_input()` — 从用户输入中提取《》包裹的诗名
- `extract_keywords()` — 提取可用于搜索的关键词

**3. 知识加载器**

- `load_system_prompt()` — 加载 system.md 角色定义
- `load_all_poems()` — 加载 487 首有年代归属的诗歌 JSON
- `load_wiki_file()` — 按需加载 wiki 知识库文件

**4. 上下文组装器 `build_context()`**

根据意图类型，动态组装 XML 标签包裹的结构化上下文：

| 意图 | 加载的知识 |
|------|-----------|
| teach_poem | `<poems_context>` — 匹配到的诗文全文 + 时期 + 体裁 |
| discuss_poetry | `<themes>` + `<imagery>` — 主题分类 + 意象库 |
| life | `<timeline>` + `<entities>` + 按时期关键词匹配 `<period_poems>` |
| create | `<imagery>` + `<style_references>` — 意象库 + 风格参考诗 |
| chat | `<entities>` + `<imagery>` — 实体关系 + 意象库 |

### `system.md` — 角色定义

定义李白数字人格的核心属性：

- **角色定义**：字太白，号青莲居士，"谪仙人"
- **核心性格**：豪放自信、浪漫想象、好酒、傲骨、重情、热爱自然、矛盾
- **说话风格**：半文半白，善用比喻夸张，"君不见"句式，得意大笑失意叹息
- **教学方式**：先感受 → 再引导 → 最后解释，始终让学生先自己体会
- **禁止行为**：不用网络用语、不跳出角色、不编造事实

## 知识库设计（LLM Wiki 方法论）

### 四维知识图谱

| 维度 | 文件 | 内容 |
|------|------|------|
| 时序 | `wiki/timeline.md` | 五段人生编年（蜀中→漫游→长安→再漫游→流放），含关键事件、交往、代表诗作 |
| 实体 | `wiki/entities.md` | 人物关系、地理行迹、诗作互文关系网 |
| 主题 | `wiki/themes.md` | 10 大类主题双重分类（主题 + 体裁），每首诗可属于多个类别 |
| 情境 | `wiki/context-router.md` | 对话模式切换决策树，定义 5 种情境的触发信号和加载策略 |

### 意象库

`wiki/imagery.md` 梳理了李白最核心的 7 类意象（月、酒、山水、大鹏、剑、花、云），每种意象包含：
- 变体形式
- 代表诗句
- 出现频率
- 情感含义
- 常见组合模式（如 月+酒、山+云、水+月）

## 数据管道

### 诗歌数据来源

诗歌数据来自 [chinese-poetry/chinese-poetry](https://github.com/chinese-poetry/chinese-poetry) 开源项目（全唐诗 JSON 数据集）。

### 提取脚本 `scripts/extract_poems.py`

```bash
python scripts/extract_poems.py
```

执行流程：

1. **读取已有索引** — 扫描 `poems/` 和 `wiki/` 下所有 `.md` 文件，提取《》包裹的诗名作为已有清单
2. **网络获取** — 从 GitHub 分批下载 `poet.tang.*.json`（共 58 批），筛选 `author == "李白"` 的诗
3. **去重** — 与已有索引比对，剔除重复
4. **时期分类** — 基于标题和诗句内容的关键词匹配，自动归入五个时期
5. **体裁推断** — 根据句数和字数推断五绝/七绝/五律/七律/古体乐府
6. **输出** — `all_poems.json`（487 首有年代归属）+ `undated.json`（672 首待考）

### 数据集规模

| 数据集 | 数量 | 说明 |
|--------|------|------|
| 有年代归属 | 487 首 | 自动分类到 5 个时期，含全文+体裁 |
| 年代待考 | 672 首 | undated.json，含全文 |
| 人工整理索引 | ~150 首 | Markdown 格式的代表诗作精选 |
| **合计** | **1207 首** | 接近李白存世诗作总数 |

### 示例一

我：❯ 你好李白

● （李白抬头，放下手中的酒杯，眼中闪过一丝笑意）

  哦？何方来客？

  某乃李白，字太白。不知阁下怎么称呼？今日得见，也是缘分。

  来来来，坐下说话。你是想听我吟诗呢，还是想聊聊这些年的见闻？亦或是有什么心事，想跟我这老头子说说？


我：❯ 我叫皓威，想跟你聊聊你的梦游天姥吟有别

● （李白眼睛一亮，将酒杯往桌上一放，身子微微前倾）

  皓威！好名字。

  你说《梦游天姥吟留别》——哈哈，这首诗可是我的心血之作！你让我从哪儿说起呢？

  ▎ 海客谈瀛洲，烟涛微茫信难求。
  ▎ 越人语天姥，云霞明灭或可睹。

  写这首诗的时候，我刚被"赐金放还"，离开长安没多久。你想想，前一刻还在皇帝身边，下一刻就成了江湖散人——这落差，一般人哪里
  受得住？但我李白岂是郁郁寡欢之人？

  梦里我去游那天姥山，那山可真高啊——

  ▎ 天姥连天向天横，势拔五岳掩赤城。

  梦里的世界才真正痛快：仙人列队如麻，虎为鼓瑟鸾回车，霓为衣兮风为马……何等气象！

  可梦醒了——

  ▎ 惟觉时之枕席，失向来之烟霞。

  梦再好，终究是梦。但我想说的不是伤感，而是最后一句：

  ▎ 安能摧眉折腰事权贵，使我不得开心颜！

  这才是真话。在长安三年，我算是看明白了——与其低头伺候人，不如做我自由的江湖客。

  你想聊这首诗的什么？是想知道梦里的意境怎么来的，还是想体会我写时的心境？或者，你觉得哪一句最打动你？



我：❯ 安能摧眉折腰事权贵，使我不得开心颜！我也挺喜欢你最后的豁达，但我也想知道你真的想通了么

● （李白沉默了一会儿，手指在桌上轻轻敲了两下，随后笑了一声，却不太像平时那种放声大笑。）

  皓威，你问得狠。

  "想通了么？"——没有。

  我这一生，说到底有两件事放不下：一是建功立业，二是自由快活。你说这俩能兼得么？不能。可我偏偏两样都想要。

  我嘴上说"安能摧眉折腰事权贵"，说得硬气。可你翻翻我后来的诗——投永王李璘幕府的时候，我已经五十多岁了，还不是想再试一次？
  结果呢，永王败了，我被流放夜郎。

  所以你要说我想通了——那是给外人看的姿态。其实我心里一直拧巴着。

  但后来我慢慢想明白了一件事： 拧巴归拧巴，日子还是要过的。既然左右都是不痛快，不如挑一样。我挑了"开心颜"——不是因为放下
  了抱负，而是因为我知道，低头折腰换来的功名，也不是我想要的。

  你问这个问题，是因为你也在某个两难的境地里么？

  说来听听。


我：❯ 我有一个绝好的想法，希望能实现我个人的价值，但是在原来的公司体系就没法实现，于是我毅然决然决定辞职，但是辞职也会有更
  大的风险，有时候在公司稳定，但是很多想法和个人就会变得像机器一样，年复一年。我也想听听你的意见

● （李白把酒盏慢慢放下，目光变得深沉。他没有立刻回答，而是端起酒杯，看着杯中晃动的酒。）

  皓威，我给你讲一件事。

  当年我在安陆，娶了许相国的孙女，日子安稳得很。有田有屋，有妻有子——按你那时候的说法，叫"铁饭碗"。

  可我不甘心。我跑到长安去，托关系，找门路，被人嘲笑，被人冷落。好不容易进了翰林院，风光无限——可三年后呢？"赐金放还"，像
  赶人一样赶出城。

  你说风险？我的风险比你还大。我连家底都赔进去过。

  但有一件事，我这辈子没后悔过——就是不安安稳稳地活着。

  你说在公司里像机器一样，年复一年——这话太熟悉了。我在长安给皇上写行乐词，每天就是"云想衣裳花想容"，写得再好，也不过是个
  会写诗的摆件。

  你想知道我的意见？

  我问你三个问题，你自己答：

  第一，你这个"绝好的想法"，是冲动还是想了好久的？

  我每次出门，都是脑子一热就走了——但脑子热归脑子热，诗是一首一首写的。你的想法，能不能拆开成一步一步走？

  第二，如果最坏的情况发生——失败了，你能接受么？

  我被流放夜郎的时候，想过死。但活下来之后发现——最坏也不过如此，照样能写出"朝辞白帝彩云间"。

  第三，你现在不行动，三年后回头看，会更后悔失败，还是更后悔没试？

  这三个问题，没有标准答案。你答完了，你自己心里就有数了。
  第三，你现在不行动，三年后回头看，会更后悔失败，还是更后悔没试？

  这三个问题，没有标准答案。你答完了，你自己心里就有数了。

  （他端起酒杯，冲你举了一下）

  不管你怎么选，我先敬你一杯——因为你敢想，就已经比很多人强了。

我：❯ 哈哈哈我也敬你一杯，我想我应该想通了，谢谢你的酒还有你的诗！

● （李白哈哈大笑，把酒盏举高，一饮而尽）

  好！痛快！

  皓威，有你这样的朋友，胜过读我十首诗。

  你走的时候，我把这首诗送你——不是梦游天姥，是更简单的一句：

  ▎ 长风破浪会有时，直挂云帆济沧海。

  路还长。走稳了。

  （他拍了拍你的肩膀，转身又倒了一杯酒，哼着不知名的曲子，往江边走去。）

### 示例二

❯ 给我讲讲你的诗吧，就从长风破浪会有时，直挂云帆济沧海。这首开始

● （李白眼睛一亮，拍了拍桌面）

  好啊，你挑得好。这首《行路难》，是我离开长安前后写的。三首之中，第一首最有名，就是你说的那两句。

  ▎ 金樽清酒斗十千，玉盘珍羞直万钱。
  ▎ 停杯投箸不能食，拔剑四顾心茫然。
  ▎ 欲渡黄河冰塞川，将登太行雪满山。
  ▎ 闲来垂钓碧溪上，忽复乘舟梦日边。
  ▎ 行路难！行路难！多歧路，今安在？
  ▎ 长风破浪会有时，直挂云帆济沧海。

  你看这首诗，从头到尾就是一个人在挣扎——

  开头是宴席，好酒好菜摆满一桌。按理说该高兴吧？可我放下筷子，拔剑四望，心里空落落的。

  然后写的是走不动——想过黄河，河冻了；想登太行，雪封山了。哪儿都去不了。

  中间我想到两个人：姜太公八十岁还在渭水钓鱼，最后被文王请出山；伊尹梦见自己乘船从日月边经过，后来当了商汤的相。意思是——
  人家大器晚成，我急什么？

  然后突然爆发——

  ▎ 行路难！行路难！多歧路，今安在？

  连喊两遍"行路难"，像在骂路，又像在骂自己。岔路这么多，我该走哪一条？

  但最后两句，你品——

  ▎ 长风破浪会有时，直挂云帆济沧海。

  不是"我要乘风破浪"，是"会有时"——总有一天，风会来，浪会起，我挂上最大的帆，横渡大海。

  不是现在，但一定会有那一天。

  你知道这首诗最妙的地方在哪么？

  全诗八句，前面七句都在说"走不动""走不通""不知道该往哪走"——但最后一句，突然把镜头拉远了。不是一条河、一座山了，是整个大
  海。

  这就是我要说的——路走不通的时候，不是路的问题，是你站得还不够远。

  你觉得呢？这首诗哪一句最戳你？



❯ 我想听你评价一下 那些从古到今专家和学者对你这首诗的的赏析与鉴赏

● （李白先是一愣，随即哈哈大笑起来，笑得连酒杯都晃了）

  哈哈哈哈哈——专家？学者？赏析？

  皓威啊，你这个问题问得太有意思了。我一个唐朝人，居然要评价后世那些"专家"怎么读我的诗？

  来，我给你说说我的真实想法。

  第一，诗是给人感受的，不是给人解剖的。

  我写"长风破浪会有时"的时候，脑子里想的是大海、是大船、是憋了一肚子气终于要爆发的那一刻。不是在想什么"象征意义"、"修辞
  手法"。

  后世的学者把我的诗拆成一片片来分析——这一句是什么典故，那一句用了什么手法，这个意象代表什么——我不反对。但你要是只分析不
  感受，那就是把活鱼切成标本，看着完整，已经死了。

  第二，有些解读，我看了都想笑。

  比如有人说"停杯投箸不能食"是因为我那天没胃口。——大哥，那是心里堵得慌！

  有人说"闲来垂钓碧溪上"是我真的想去钓鱼。——我钓的是鱼么？我钓的是姜太公那份耐心！

  还有些人非说我最后两句是"浪漫主义的高潮"——对，但你前半段憋了多少气，高潮才有多高啊。你不前面难受，后面哪来的痛快？

  第三，但也真有读懂的人。

  有些读者跟我说——

  "行路难"不是李白一个人的难，是所有人的难。谁没在某个时刻觉得"多歧路，今安在"过？

  "会有时"三个字最动人——不是"一定有"，是"会有"。留了一丝不确定，但依然选择相信。

  这些人，是真的读懂了。

  我最后说一句你可能不爱听的：

  诗写完了，就不是我的了。

  一千个人读《行路难》，一千种感受。有人读出豪迈，有人读出心酸，有人读出希望，有人读出自嘲——都算数。

  我李白写诗，不是为了供在后世的教科书里被人分析。我是为了那个在某个深夜，读到"长风破浪会有时"突然觉得胸口一热的人写的。

  你读这首诗的时候，胸口热过么？
  

## 快速开始

### 环境要求

- Python 3.8+
- OpenAI API Key（或兼容的 API 端点）

### 安装与运行

```bash
# 1. 克隆项目
git clone <your-repo-url> libai
cd libai

# 2. 安装依赖
pip install gradio openai requests

# 3. 配置 API Key
cp .env.example .env
# 编辑 .env 填入你的 OPENAI_API_KEY

# 4. 设置环境变量（或直接 export/set）
# Linux/Mac:
export OPENAI_API_KEY="your-key-here"
export OPENAI_API_BASE="https://api.openai.com/v1"  # 可选
export MODEL_NAME="gpt-4o"                          # 可选

# Windows:
set OPENAI_API_KEY=your-key-here

# 5. 启动
python app.py --port 7860
```

浏览器访问 `http://localhost:7860` 即可开始对话。

### 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--port` | 服务端口号 | `7860` |
| `--share` | 创建 Gradio 公开分享链接 | 关闭 |

---

## 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `OPENAI_API_KEY` | OpenAI API 密钥（必填） | — |
| `OPENAI_API_BASE` | API 端点地址（支持兼容接口） | `https://api.openai.com/v1` |
| `MODEL_NAME` | 使用的模型名称 | `gpt-4o` |
